# Domain Docs

Single-context repo.

Read these before domain-sensitive engineering work:

- `CONTEXT.md`
- relevant `doc/*.md`
- relevant `prd/*.md`
- relevant ADRs under `docs/adr/`

## Rules

- Use terms from `CONTEXT.md`.
- Treat `doc/current.md` as shipped behavior.
- Treat `doc/roadmap.md` as first-release target.
- Treat each `prd/*.md` as one workstream owner.
- If an ADR conflicts with a proposed change, call out the conflict.
